# 投資組合管理工具 — 開發文件 (v3.3.1)

> 這份文件的用途：讓任何一個 Claude 對話（或開發者）在沒有歷史聊天記錄的情況下，
> 也能完整理解這個專案的設計邏輯、現有功能、和未來方向，直接接手開發。

## 專案概述

純 HTML 單檔的個人投資組合管理工具，不需要安裝任何環境，用瀏覽器直接開啟即可運作。
資料透過 CSV 匯入匯出持久保存，可用 Excel 開啟或編輯 CSV 檔。

**目標使用者**：個人投資者，同時持有台股與美股，通常 5–15 檔標的。

## 核心設計理念（v3.0 重要更新）

### 雙資金池模型

工具以「**台股池（TWD）**」和「**美股池（USD）**」兩個獨立資金池來管理投資。
這個模型對應真實世界 — 資金從可動用資金（銀行）轉入券商成為「投資池」，
標的買入從池扣款，賣出與股息回流到對應池。

```
[可動用資金 TWD] ←─ (工具不管理，只是來源概念)
       ↓ ↑
[台股池 TWD]      [美股池 USD]    (兩池獨立，各自原幣管理)
       ↓ ↑                ↓ ↑
   [台股持倉]         [美股持倉]
```

**工具只管兩個投資池的活動，不管理可動用資金、池間轉移、新資金移入紀錄**
（這些由銀行/券商記錄即可，未來有 user story 再做）。

### 原幣別呈現

持倉表格、批次明細、賣出紀錄、股息紀錄全部以**原幣別**顯示。台股用 TWD，美股用 USD，不做即時換算。
只有「合併檢視」模式才會用設定匯率將美股換算為 TWD 加總。

### 主題與顏色語意（v3.1）

**Dark / Light / System 三段切換**：標題列右側可切換，預設跟隨系統設定。
所有顏色透過 CSS variables 定義，切換主題即時生效，不需重新整理。

**顏色語意（亞洲市場慣例）：**
- 🔴 **紅色** = 正向 / 上漲 / 獲利 / 過熱
- 🟢 **綠色** = 負向 / 下跌 / 虧損 / 超賣
- 🟣 **紫色** = 強調 / 選取 / 主要操作

中性元素（標籤、邊框、次要文字）一律使用灰階階層，避免顏色過載。

## 功能清單

### 持倉管理
- 新增買入（支援同標的分批）
- 同名標的合併顯示，可展開查看每批明細
- 編輯、刪除批次或整個標的
- 一鍵更新某標的的現價（同步所有批次）
- 以原幣別管理（TWD / USD），不混合計算

### 操作邏輯（先選取，再操作）
- 點列選取標的或批次（再點取消）
- 工具列按鈕根據選取狀態自動 enable/disable：
  - 未選取：只有「＋新增買入」可用
  - 選取標的：「更新現價」「加碼」「配息」「刪除整個標的」
  - 選取批次：「更新現價」「編輯」「賣出」「刪除批次」

### 資金池管理
- 設定台股池總額（TWD）與美股池總額（USD）
- 自動計算閒置現金 = 池總額 - 已投入成本 + 已實現損益 + 已收股息
- 「分池檢視 / 合併檢視」切換
- 標的的「佔池比」基於對應池總額計算

### 股息記錄
- 點選標的後，從工具列「配息」按鈕記錄
- 紀錄欄位：每股股息、股數、稅前總額（自動）、實領金額、除息日、備註
- 獨立「股息歷史」頁籤，按時間倒序顯示
- 已收股息會回流到對應池的閒置現金

### 賣出與已實現損益
- 對每批持倉操作賣出（部分或全部）
- 賣出後該批移入「交易歷史」頁籤
- 賣出損益不含股息（兩者獨立記錄，更清晰）
- 賣出回收金額會回流到對應池

### 績效計算
- 未實現損益（金額 + 報酬率 %）
- 累計股息收益
- 總報酬 = 未實現損益 + 股息
- 年化報酬率（持倉與已賣出各自獨立計算）
- 標的佔池比（同幣別池內的資金佔比）

### 警示
- 過熱閾值（預設 ≥+20%）：建議減碼
- 超賣閾值（預設 ≤-15%）：建議加碼
- 閾值可在設定頁調整

### 資料
- CSV 匯出/匯入（包含池設定、買入、賣出、股息四個區塊）
- CSV 可用 Excel 開啟編輯

## 計算公式

### 持倉成本與損益（原幣）
```
總成本     = 股數 × 買入均價 + 買入手續費
市值       = 股數 × 現價
未實現損益 = 市值 - 總成本
報酬率 %   = 未實現損益 / 總成本 × 100   ← 真實報酬率（含手續費影響，與券商對帳單一致）
```

### 賣出損益（含手續費和交易稅）
```
實領金額   = 股數 × 賣出價 - 手續費 - 交易稅
買入成本   = 股數 × 買入均價 + 該批次手續費（按股數比例攤分）
已實現損益 = 實領金額 - 買入成本
報酬率 %   = 已實現損益 / 買入成本 × 100
```

### 同標的合併
```
均攤成本 = Σ(各批次股數 × 各批次買入價) / 總股數
加權天數 = Σ(各批次股數 × 各批次持有天數) / 總股數
```

### 年化報酬
```
持倉中：年化 = ((市值 + 已收股息) / 總成本) ^ (365 / 加權天數) - 1
已賣出：年化 = (賣出總額 / 買入總成本) ^ (365 / 持有天數) - 1
       （賣出年化不含股息，股息獨立累計）
※ 持有天數最少以 1 天計算
```

**短期持有警示**：當持有天數 < 30 天時，年化報酬會用淡灰色 + ⚠ 圖示顯示，
並在 hover 時提示「年化僅為數學推算，短期內參考價值有限」。

這是因為短期年化會把小報酬複利放大成天文數字，例如持有 4 天 +2.69% 的年化會是 +1029%。
數字本身計算正確，但意義上不應該作為決策依據。

### 資金池
```
池總額         = 使用者設定
閒置現金       = 池總額 - 已投入成本 + 已實現損益 + 已收股息
標的佔池比     = 該標的成本 / 該幣別池總額 × 100
```

### 真實總報酬（v3.1+ 新增）
這是衡量整體投資績效的關鍵指標，回答「這個池子總共賺了多少」。

```
真實總報酬     = 未實現損益 + 已實現損益 + 已收股息
累計投入成本   = 目前持倉成本 + 已賣出對應的買入成本
真實總報酬率   = 真實總報酬 / 累計投入成本 × 100
```

範例：池子初始 50萬，買進 30萬，目前市值 35萬，賣出獲利 2萬，股息 5千
- 未實現 = 35萬 - (30萬 - 賣出對應成本) = 7萬
- 已實現 = 2萬
- 股息 = 5千
- 真實總報酬 = 9.5萬
- 累計投入成本 = 目前持倉成本 + 已賣出對應成本

### 合併檢視（換算 TWD）
```
合併池總額   = TWD池 + USD池 × 匯率
合併已投入   = TWD已投入 + USD已投入 × 匯率
（其他項目同理）
```

## CSV 格式

匯出的 CSV 包含四個區塊，用標頭分隔：

### [POOLS] 池設定
```
poolTWD,poolUSD,usdRate
500000,10000,32
```

### [BUYS] 買入紀錄
```
id,name,ticker,currency,shares,costPerShare,buyFee,currentPrice,buyDate
1,台積電,2330,TWD,100,820,140,950,2024-06-15
```

| 欄位 | 說明 |
|------|------|
| id | 唯一識別碼（程式產生，不要手動改） |
| currency | TWD 或 USD |
| shares | 持有股數（賣出後為剩餘） |
| costPerShare | 該批買入均價（原幣，不含手續費） |
| buyFee | 該批買入手續費（原幣，部分賣出後會按比例減少） |
| currentPrice | 目前現價（原幣，會被覆蓋） |

### [SELLS] 賣出紀錄
```
id,name,ticker,currency,shares,costPerShare,sellPrice,buyFee,sellFee,sellTax,buyDate,sellDate,holdDays,realizedPnL
```

| 欄位 | 說明 |
|------|------|
| buyFee | 賣出時當批攤分的買入手續費 |
| sellFee | 賣出手續費 |
| sellTax | 交易稅（台股、美股皆適用） |
| realizedPnL | 已實現損益（已扣除所有費用稅金） |

**向下相容**：匯入時支援三種格式
- 13 欄（最早版本，沒有手續費/稅）
- 14 欄（目前版本）
- 16 欄（中間版本含 returnPct + annualized，匯入時自動忽略）

報酬率與年化報酬不再儲存於資料中，每次顯示時即時重算，確保資料一致性。

### [DIVIDENDS] 股息紀錄
```
id,name,ticker,currency,perShare,shares,grossAmount,netAmount,exDate,note
1,富邦金,2881,TWD,3.5,500,1750.00,1662.50,2024-07-15,扣繳5%
```

| 欄位 | 說明 |
|------|------|
| perShare | 每股股息（原幣） |
| grossAmount | 稅前總額（perShare × shares） |
| netAmount | 實領金額（含扣稅後） |
| exDate | 除息日 |
| note | 備註（CSV 中逗號會被替換為分號） |

## 資料儲存機制（v3.1+）

工具使用「localStorage 快取 + CSV 主資料」的雙層架構：

### 工作模式
- **「📂 開啟資料」** — 從 CSV 檔案載入，會覆蓋目前資料
- **「💾 儲存資料」** — 把目前資料下載成 CSV 檔案
- **localStorage** — 編輯時自動同步寫入瀏覽器，作為快取與防遺失

### 跨裝置使用建議流程

把 HTML 和 CSV 都放在 iCloud Drive / Google Drive 等同步資料夾：

1. **任何裝置打開 HTML** → 點「開啟資料」載入最新 CSV
2. **編輯資料** → 變動會即時存進 localStorage（不怕當機）
3. **離開前點「儲存資料」** → 下載並覆蓋 CSV，雲端同步到其他裝置

### 未儲存提示

- 編輯後標題列顯示：「● 有變動尚未儲存到資料檔案」（紅色）
- 「儲存資料」按鈕會出現紅色脈動框
- 關閉頁面時瀏覽器會跳警告

### 重要注意事項

- **避免兩台裝置同時編輯**，無法處理同步衝突
- 同樣的，**避免一台裝置開兩個視窗同時編輯**
- 若不小心發生衝突，**先儲存的會被後儲存的覆蓋**

### 自動儲存的安全機制

- localStorage 只在使用者明確操作後才寫入（避免初始化時誤覆蓋資料）
- 偵測到 localStorage 資料異常時，會跳警告而不是默默載入範例
- 「重設為範例資料」「清空所有資料」前都會二次確認

## 技術說明（給開發用）

- 純 HTML + CSS + Vanilla JavaScript，無外部依賴（字型除外）
- 字型：Google Fonts（DM Mono + Syne）
- 狀態存在 JS 變數：
  - `buys[]` — 買入批次
  - `sells[]` — 已實現賣出紀錄
  - `dividends[]` — 股息發放紀錄
  - `sel` — 目前選取狀態 `{type:'group'|'batch', ticker, batchId?}`
  - `expanded` — 已展開的 ticker Set
  - `viewMode` — 'split' | 'merged'
- `render()` 為主要 UI 更新函式，所有資料變動後呼叫
- `renderTable()` 只更新表格，用於選取/展開時減少重繪
- 四個 Modal：buyModal（買/編輯/加碼）、sellModal、priceModal、dividendModal
- `groups()` 將 buys 依 ticker 合併為標的群組，計算各種衍生數據
- `poolStats(currency)` 計算單一池的指標

| 版號位置 | 觸發條件 | 範例 |
|----------|----------|------|
